# 马尼拉（Manila）—— 联机桌游项目

## 项目概述

马尼拉是一款以 19 世纪菲律宾港口贸易为背景的轻策桌游的电子化实现。玩家扮演商人，通过竞拍港务长、在货船上放置帮手、操纵船运获得利润，最终以现金和股票总价值决定胜负。本项目实现了完整的游戏规则引擎，并支持在线多人联机对战。

## 核心功能与亮点

### 1. 完整的游戏规则引擎

覆盖马尼拉桌游全部阶段，采用**阶段状态机**驱动：

| 阶段 | 说明 | 引擎 |
|------|------|------|
| 🔨 **竞拍港务长** | 玩家轮流竞拍，港务长获得权力与股份 | `AuctionPhase` |
| ⚙️ **港务长设置** | 港务长选择货物、设定船起始位置 | `SetupPhase` |
| 🧩 **放置帮手** | 所有玩家将帮手放到船/港口/船坞/保险等位置 | `PlacementPhase` |
| 🎲 **移动** | 港务长掷骰决定货船前进格数 | `MovementPhase` |
| 🏴‍☠️ **海盗检查** | 海盗船长决定拦劫船只 | `PirateEngine` |
| 🧭 **领航员调整** | 领航员可微调船位置 | `PilotEngine` |
| 💰 **利润分配** | 自动结算收入与支出 | `ProfitPhase` |
| 📈 **价格上涨** | 成功靠港的货物价格上涨，触发游戏结束条件 | `PricePhase` |

**亮点**：
- **服务器权威架构**：所有游戏逻辑在服务端执行，客户端仅接收状态，杜绝作弊
- **静态 ActionValidator**：客户端通过 `ActionValidator.getValidActions()` 获取有效动作列表，无需实例化引擎
- **自动化阶段流转**：利润分配与价格上涨等阶段自动执行，无需玩家操作

### 2. 在线多人联机

- **Socket.IO 实时同步**：WebSocket 全双工通信，低延迟
- **房间管理系统**：创建房间、加入房间、玩家列表更新
- **断开处理**：玩家断线自动清理房间，其他玩家收到提示（React 模态框）
- **单例连接**：`getSocket()` 确保 Socket 连接在页面间复用，避免重复认证

### 3. 多种游戏模式

- **本地模式**：单机多玩家轮流操作，适合本地教学或测试
- **联机模式**：通过网络与其他玩家对战

### 4. 结算与财务系统

- **航程结算弹窗**：每轮结束后显示详细收支明细（货舱分红、港口收入、保险赔付、海盗掠夺等）
- **股票系统**：购买、持有、抵押、赎回股票
- **贷款机制**：以股票为抵押获取现金
- **最终计分**：现金 + 股票价值 - 抵押罚款 = 总分

### 5. 本地化与 UI/UX

- **中文界面**：所有游戏文本中文化
- **阶段提示**：右侧面板显示当前阶段说明
- **操作面板**：根据游戏阶段动态切换操作界面（7 种面板）
- **联机状态**：清晰显示"你的回合" / "等待对手操作..."
- **游戏日志**：实时显示操作历史

## 数据流

### 联机模式

```
LobbyPage                     GamePage (在线)
    │                              │
    │  room:create/join             │  game:action ──────┐
    │      │                       │                    │
    │      ▼                       │                    ▼
    │  room:joined  ◄──────────────┤            Server GameEngine
    │      │                       │                    │
    │      ▼                       │                    ▼
    │  game:started ──────► game:state ◄────── io.to(room).emit
    │                              │
    │                              │  game:ended → 模态框 → /lobby
```

### 核心设计原则

- **服务端权威**：`GameEngine` 仅在服务端运行，客户端不持有引擎
- **状态无状态化**：完整游戏状态通过 Socket.IO 传输，客户端仅展示
- **单例 Socket**：连接在页面切换时保持，避免重复连接

## 规则特色

- **3-5 名玩家**：支持 3 到 5 人对战
- **4 种货物**：翡翠、丝绸、香料、瓷器，各有不同成本和收益
- **动态船位**：港务长可调整货船起始位置
- **领航员系统**：小额/大额两种下注方式
- **保险公司**：可选投保，船失事时获赔
- **盲流（偷渡）**：特殊机制允许免费放置帮手
